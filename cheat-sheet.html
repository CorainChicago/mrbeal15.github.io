Accessing elements in Arrays, Hashes, and Objects

Iterating over data structures is something that we have had to do a lot, and when dealing with multiple languages it's handy to have a one-stop-shop for the different ways to iterate and access data in different structures

Iterating Over Arrays in Ruby
One of the most common methods used to iterate over structures in Ruby is the each method:
array.each do |x|
array.reverse
end
This takes each element in the array assigns it a placeholder of 'x', and then executes the block in the each method. In this instance, it is taking every element in the array and reversing it.
Iterating Over Hashes in Ruby
Just like arrays, Hashes have an each method as well that allows us to manipulate both the key and the value in an hash
hash.each do |k, v|
k.upcase
v.downcase
end
This takes each key in the hash and gives it a placeholder 'k', and then takes each key's values and gives them a placeholder 'v'. It then executes the block of code on the key, value or both.

When using the each method, you can either put the block of code on one sinlge line, or use multiple lines like above. If you use one line, you can place everything inside curly braces like: array.each {|x| x.upcase}

Another common way to iterate is to use the map/map! methods. These methods take the elements in the original array and copies them into a new array. The destructive version of the map method, map!, takes those elements and replaces the original array.

Iterating over an Array in JavaScript
Javascript doesn't have a built in function like Ruby, so iterating over arrays takes a little more effort, but here is an effective way to do it
while(i=0; i < array.length; i++){
array[i].reverse();
}
This uses a counter to go through each element in the array, and for each element it adds 1 to the counter. It will continue going through the array as long as the counter, 'i', is less than the array length.
JavaScript also has objects, that resemble Ruby hashes in that they have properties and values. The properties and values of objects can be accessed, regardless of how nested the properties are.
friends = {
Jim: {age: 26, school: "OU"},
Steve: {age: 30, school: "OSU"},
"Kim": {age: 22, "school": "Tulsa"},
"Carla": {age: 25, "school": "ORU"}
}
To access Steve's school using dot notation: friends.Steve.school
To access Carla's school using bracket notation: friends["Carla"]["age"]
